# 검은 화면 및 성능 최적화판

## 검은 화면 방지

- 세로 화면에서 전체 게임 컨테이너를 강제로 회전하던 CSS 제거
- Canvas 크기를 최소 320×180 이상으로 안정화
- 배경 이미지 실패 시 그라데이션 대체 화면 표시
- 이미지 로딩 실패와 JavaScript 오류를 사용자 화면에 표시
- 오류 화면에서 서비스워커를 해제하고 다시 불러올 수 있음
- HTML·JS·CSS는 Network First 캐시 전략 적용

## 성능 최적화

- 모바일 DPR 최대 1.35, 데스크톱 최대 1.75
- 화면 좌표로 변환된 경로를 resize 시 한 번만 캐시
- HUD 갱신을 약 120ms 간격으로 제한
- 백그라운드 탭에서는 렌더링과 업데이트 중단
- 이미지 비동기 디코딩 적용

## 모바일 가로 모드

- PWA manifest와 Screen Orientation API의 가로 모드 요청은 유지
- 세로 상태에서는 회전 안내를 표시하며, 검은 화면을 유발하던 CSS 90도 회전은 사용하지 않음
